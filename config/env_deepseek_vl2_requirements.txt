# Isolated environment for the DeepSeek-VL2 adapter ONLY.
#
# DeepSeek-VL2 is not natively supported by `transformers`; it loads through
# DeepSeek's own `deepseek_vl2` package (github.com/deepseek-ai/DeepSeek-VL2),
# whose pyproject pins its stack EXACTLY (torch==2.0.1, transformers==4.38.2,
# xformers, timm, attrdict). Those pins are load-bearing — the modeling code
# imports symbols removed from newer transformers — so this file adds NO
# competing version constraints: the package drives the stack (panel F-01/
# F-03: an added torch floor made the file pip-unsatisfiable).
#
# KNOWN WORKAROUNDS (panel F-02, verified against upstream issues):
# - `attrdict` (transitive dep) is abandoned and import-broken on
#   Python >= 3.10. Build this venv with Python 3.9, OR install `attrdict3`
#   FIRST (line below) so the import resolves before deepseek_vl2 lands.
# - torch==2.0.1 has no Blackwell support: this lane REQUIRES an
#   Ampere-class rental (A100 PCIe per the M9 plan), not the RTX PRO 6000.
#
# The attrdict fix-up runs AFTER this file installs (see
# scripts_cloud_setup_b2.sh: uninstall attrdict, install attrdict3) because
# both distributions ship the same `attrdict` module and install order via
# pip is not guaranteed.
#
# The exact `pip freeze` of the venv actually used is captured into the run
# evidence at rental time (these lines are the inputs, not the record).

deepseek_vl2 @ git+https://github.com/deepseek-ai/DeepSeek-VL2.git
